# 예측 전략과 통계 분석

⚠️ 먼저 짚고 가기: 로또 6/45의 각 회차는 **독립 시행**이며, 과거 데이터는 다음 회차 확률에 영향을 주지 않습니다. 이 문서의 전략들은 "과거 분포와 어긋나지 않는 번호 세트를 합리적으로 생성"하는 도구이지 당첨 확률을 높이는 것이 아닙니다.

## 통계 지표 (analyze 함수)

데이터 N회차가 주어질 때 계산하는 값들.

### 1. 전체 빈도 (`freq_all`)
번호 1~45 각각이 전체 기간 동안 몇 번 당첨번호로 나왔는지. 균등 분포라면 기대값은 `N × 6 / 45 ≈ 0.133 × N`. 실제로는 미세한 편차가 있습니다.

### 2. 최근 빈도 (`freq_recent`)
최근 `RECENT_WINDOW`회 (기본 50회)만 대상으로 한 빈도. "요즘 자주 나오는 번호"를 보는 용도.

### 3. 쌍 동반 출현 (`pair_count`)
번호 쌍 `(a, b)`가 같은 회차에 함께 뽑힌 횟수. 한 회차에서 6개 중 2개를 고르는 조합은 C(6, 2) = 15개이므로, N회차 전체에서 `15 × N` 쌍 사건이 누적됩니다.

### 4. 갭 (`gap`)
`latest_round - (번호가 마지막으로 나온 회차)`. 오래 안 나온 번호를 식별.

### 5. 합계 (`sum_*`)
한 회차 6개 번호 합의 평균 (`avg_sum`)과 권장 범위.
권장 범위는 단순히 `avg ± 35`로 정의 — 과거 대부분의 회차가 이 안에 들어옵니다.

이론적 기대값은 `6 × (1+45)/2 / 2 = 138`. 실제 평균도 이 근방.

### 6. 홀수 최빈값 (`odd_mode`)
한 회차 6개 번호 중 홀수 개수의 최빈값. 보통 3개입니다 (홀·짝 비율이 대칭이므로).

### 7. 구간 분포 (`range_dist`)
1–10, 11–20, …, 41–45의 5개 구간별 누적 출현 횟수.

## 5가지 전략

### ① 전체빈도 가중 (`strategy_frequency`)

핫넘버(많이 나온 번호)에 더 큰 확률. 각 번호 n의 가중치 = `freq_all[n]`.

```
P(n 선택) ∝ freq_all[n]
```

`weighted_sample`로 중복 없이 6개 추출.

**특징**: 역사적으로 빈도 높은 번호가 자주 뽑힘. 엄밀히는 균등분포에서 약간 벗어나지만 실용상 차이 작음.

### ② 최근트렌드 가중 (`strategy_recent`)

최근 `RECENT_WINDOW`회차 빈도를 가중치로. 단, 최근에 한 번도 안 나온 번호도 후보에서 제외되지 않도록 +1 smoothing.

```
P(n 선택) ∝ freq_recent[n] + 1
```

**특징**: "요즘 번호"에 편향된 세트. `RECENT_WINDOW`가 작을수록 편향이 커집니다.

### ③ 콜드넘버 반등 (`strategy_cold`)

오래 안 나온 번호를 선호. 가중치 = 갭(마지막 출현 후 경과 회차) + 1.

```
P(n 선택) ∝ gap[n] + 1
```

**특징**: 갬블러의 오류(Gambler's fallacy)를 체계화한 버전. 확률을 실제로 높이지는 않지만, 다른 전략과 상보적인 선택지를 만듭니다.

### ④ 동반출현 쌍기반 (`strategy_pair`)

가장 자주 함께 나온 상위 20개 쌍 중 하나를 랜덤 선택하여 "씨앗 쌍"으로 삼고, 그 쌍과 함께 자주 나온 다른 번호들로 나머지 4개를 채웁니다.

알고리즘:
```
1. top_pairs = pair_count 상위 20개
2. seed = random.choice(top_pairs)  → [a, b]
3. picked = [a, b]
4. while len(picked) < 6:
     partners[x] = Σ pair_count[(p, x)]   (p ∈ picked, x ∉ picked)
     다음 번호를 partners 가중치로 샘플링
```

**특징**: 숫자 간 "친화성"을 반영. 같은 번호들이 반복될 가능성이 가장 높은 전략.

### ⑤ 균형형 (`strategy_balanced`)

전체빈도 가중으로 뽑되, **역사적 분포 제약**을 만족할 때까지 재시도 (최대 500회).

제약 조건 (`is_balanced`):
1. **합계**: `sum_min ≤ 합 ≤ sum_max` (평균 ± 35)
2. **홀수 개수**: 0개나 6개가 아닐 것 (= 1~5개)
3. **구간 편중**: 10단위 구간 중 한 곳에 5개 이상 몰리지 않을 것
4. **연속 번호**: 4개 이상 연속된 번호(예: 12, 13, 14, 15) 배제

```python
for _ in range(500):
    nums = weighted_sample({n: c for n, c in freq_all.items()})
    if is_balanced(nums, stats):
        return nums
return nums   # 실패 시 마지막 시도 반환
```

**특징**: 실제 당첨 번호의 "느낌"에 가장 가까운 세트 생성. 보통 몇 번 재시도 안에 통과.

## 가중치 샘플링 (`weighted_sample`)

중복 없이 6개를 뽑는 방식:

```
1. 각 번호 n에 대해 가중치 w[n] = max(weights.get(n, 0), 0.001)
2. 반복 6회:
     pick = random.choices(nums, weights=w, k=1)[0]  # 가중치 확률로 1개
     if pick not in picked:
         picked.append(pick)
         w[해당 인덱스] = 0     # 다시 뽑히지 않도록
```

0으로 막는 방식이므로, 가중치가 모두 0이어도 (이론적으로만 가능) 초기 `0.001` floor 덕분에 무한 루프를 피합니다.

Python의 `random.choices`와 JavaScript의 누적합 방식이 다르지만, 결과 분포는 동일합니다.
